Sources & Methodology

How pseudosec. collects and presents information — and what you should know before relying on it.

⚠

Educational use only. pseudosec. is an independently run awareness project. It is not affiliated with the Australian Government, ACSC, or any security vendor. It is not a real-time security operations tool and should not be used as a substitute for professional security advice or official government guidance.

What pseudosec. is

pseudosec. aggregates publicly available Australian cyber security information into a single, readable view. It is designed for general awareness — helping everyday Australians and those new to cyber security understand the threat landscape without needing to monitor multiple government and industry sources themselves.

It is a personal project. There is no team, no 24/7 monitoring, and no service level agreement. Feeds may experience delays. Summaries may contain errors. Content is curated with care but is not professionally fact-checked before publication.

Data sources

All information is drawn from publicly available sources. No proprietary threat intelligence feeds are used.

Australian Signals Directorate — Alerts & Advisories
Australia's national cyber authority. ASD advisories and alerts are the primary source for threat activity affecting Australian organisations and individuals. Chosen for authority and Australian relevance.
Every 30 min RSS feed
Scamwatch — ACCC
The ACCC's national scam reporting platform. Used for the live scam alert strip. Scam alerts are pulled from the public Scamwatch feed and displayed as-is without editorial modification.
Every 30 min RSS feed
CISA Known Exploited Vulnerabilities (KEV)
CISA's authoritative catalog of vulnerabilities being actively exploited in real attacks. Used for the CVE panel. Chosen because KEV is the most operationally relevant vulnerability list — inclusion requires confirmed active exploitation.
Daily JSON API
Australian cybersecurity news
Aggregated from multiple Australian and internationally relevant security news sources via RSS. Articles are deduplicated and sorted by recency. Sources include ASD, iTnews, SC Media Australia, and similar publications.
Every 30 min RSS feed
ACSC Annual Cyber Threat Report
The statistics displayed in the live tracker (incident counts, average costs, hotline calls) are sourced from the most recent ACSC Annual Cyber Threat Report. These figures are static and updated manually when a new report is published.
Annual Manual
HaveIBeenPwned — Troy Hunt
Used on the Resources page breach checker only. Email addresses entered by users are sent directly to the HIBP API and are not stored, logged, or processed by pseudosec. HIBP is run by Australian security researcher Troy Hunt.
Real-time API (Resources only)
Open-Meteo
Used for the weather widget in the header. Location is requested from the browser and sent directly to Open-Meteo's public API. No location data is stored by pseudosec.
Real-time API

AI Briefing

The daily AI Briefing is generated using the Claude API (Anthropic). It summarises recent threat activity drawn from the news and advisory feeds.

⚠ The briefing is not fact-checked by a human before publication. AI summaries can misinterpret, omit, or inaccurately represent source content. Treat it as a starting point — always verify significant information against original sources.
⚠ The briefing is generated once daily. It does not reflect real-time events and may be outdated by the time you read it.
ℹ Anthropic's data usage policies apply to content sent to the Claude API. pseudosec. does not send personally identifiable information to the API.

Limitations

— This is a single-person project. There is no team monitoring feeds, reviewing AI output, or responding to incidents around the clock.
— Feed ingestion relies on external RSS sources and APIs. Delays, outages, or changes to upstream sources may result in stale or missing data.
— AI-generated summaries may contain factual errors, hallucinations, or outdated information. Do not use the AI Briefing as your sole source of threat information.
— pseudosec. does not generate original threat intelligence. All content is drawn from publicly available sources. It is a synthesis and presentation layer, not an intelligence product.
— For specific cyber security advice, contact a qualified security professional, the Australian Signals Directorate, or call the ACSC hotline on 1300 CYBER1.
